# Context Refresh Requirement

**Defined by Yoshimi Nakane**

## Definition

Context Refresh Requirement is the requirement to verify, before an AI-assisted decision becomes executable, whether the conditions that made the original approval legitimate still hold.

If those conditions have changed, the prior approval is stale and should not be treated as sufficient authority for execution.

## Core Claim

Execution should not inherit legitimacy from an approval whose context has changed.

## Positioning

Context Refresh Requirement sits between authority formation and execution.

It asks whether the human, contextual, institutional, and responsibility conditions that supported a prior approval remain sufficiently intact at the moment an AI-assisted decision or action becomes consequential.

## Relationship to Authority Drift

Authority Drift can occur when recorded or assigned authority is carried forward without verifying whether the conditions that made that authority legitimate still hold.

Context Refresh Requirement is one mechanism for detecting, surfacing, or preventing stale authority before irreversible execution.

## Relationship to Legitimacy Layer

The Legitimacy Layer defines when human judgment can count as valid authority before execution.

Context Refresh Requirement asks whether that legitimacy remains intact at the point where execution is about to occur.
