# Context Refresh Requirement

## Definition
Before an AI-assisted decision becomes executable, the system must verify whether the conditions that made the original approval legitimate still hold.

If those conditions have changed, the prior approval is stale and should not be treated as sufficient authority for execution.

## Positioning
This requirement operates as an upstream safeguard between legitimacy and execution.

It ensures that authority is not assumed to persist automatically over time.

## Relationship to Authority Drift
Context Refresh Requirement is the preventive condition.

Authority Drift is the failure mode when this condition is not applied.
